# Guideline Visual

## Objetivo
Centralizar referências visuais da interface (mockups e capturas) para orientar evolução de layout sem espalhar arquivos em múltiplas pastas.

## Estrutura
- `mocks/`: imagens de referência consolidadas das pastas antigas.

## Fonte dos mocks
- Origem 1: `docs/mock-up`
- Origem 2: `docs/PRINT-MARTLIST`

## Regras de uso
- Sempre consultar os mocks antes de ajustes grandes de layout.
- Não editar os arquivos originais em `docs/`.
- Novos mockups devem ser adicionados em `docs2/guideline_visual/mocks/` com nome descritivo.
